## Phase 1: Foundation (InitOptions + CoreDeps)

**Goal:** Extract `InitOptions` and `CoreDeps` into standalone files so they can be imported independently. Add `supabaseClient` to `CoreDeps` and a static `create()` factory that absorbs lines 362-461 of the current `app_initializer.dart`.

**Agent:** `general-purpose`

### Sub-phase 1.1: Create `init_options.dart`

**File:** `lib/core/di/init_options.dart` (NEW)
**Why:** `AppInitializer.initialize()` currently takes `logDirOverride` as a loose string param. `main_driver.dart` needs `isDriverMode` and test overrides. Bundling these into a typed options class removes positional ambiguity and makes the init contract explicit.
**Time:** 2 min

Create this file with the following exact content:

```dart
import 'package:supabase_flutter/supabase_flutter.dart';

import 'package:construction_inspector/services/photo_service.dart';

/// Configuration options passed to [AppInitializer.initialize].
///
/// WHY: Replaces loose parameters (logDirOverride, isDriverMode) with a typed
/// contract. Enables test/driver overrides without modifying the initializer.
class InitOptions {
  /// When true, substitutes TestPhotoService and skips real camera access.
  final bool isDriverMode;

  /// Override for debug log directory (used by integration tests).
  /// Empty string = use default from PreferencesService.
  final String logDirOverride;

  /// Inject a pre-configured SupabaseClient (for testing or driver mode).
  /// When non-null, skips Supabase.initialize() and uses this client directly.
  final SupabaseClient? supabaseClientOverride;

  /// Inject a custom PhotoService (for testing or driver mode).
  /// When non-null, used instead of the default PhotoService(photoRepository).
  final PhotoService? photoServiceOverride;

  const InitOptions({
    this.isDriverMode = false,
    this.logDirOverride = '',
    this.supabaseClientOverride,
    this.photoServiceOverride,
  });
}
```

**Verify:** File compiles - no downstream consumers yet, so no analysis errors expected.
