## Code Review: jcodemunch-mcp Dart Enhancement

### Summary
Solid, well-documented Dart language support across 17 files. Strong domain knowledge (Dart 3 features, generated code filtering, package URI resolution) and good defensive practices. Two high-severity issues and seven medium-severity issues found.

---

### High Severity

1. **H1. Potential ReDoS in `_DART_CONDITIONAL_IMPORT_RE`** ‚Äî `imports.py:274-276`
   - `(?:\s*(?:as\s+\w+|show\s+[\w\s,]+|hide\s+[\w\s,]+))*\s*;` has nested quantifiers. `[\w\s,]+` inside `(?:...)*` creates polynomial backtracking on malformed input without `;`.
   - **Fix**: Replace `[\w\s,]+` with `[^;]+?` or simplify to just capture the two URIs.

2. **H2. `_resolve_dart_package_import` O(n) linear scan per specifier** ‚Äî `imports.py:320-324`
   - Iterates all `source_files` with normalization per `package:` import. O(imports √ó files) in large monorepos.
   - **Fix**: Pre-build a normalized path dict (like `_get_sql_stems`).

---

### Medium Severity

3. **M1. DRY violation: 3 copy-pasted `has_const_or_final` blocks** ‚Äî `extractor.py:1072-1114, 1166-1181`
   - **Fix**: Extract `_has_dart_const_or_final(node) -> bool`.

4. **M2. `as` alias not captured in import names** ‚Äî `imports.py:253-260`
   - `import 'x.dart' as foo;` produces empty names; JS captures aliases.
   - **Fix**: Add capture group for `as` identifier.

5. **M3. `show` + `hide` combination not handled** ‚Äî `imports.py:257`
   - Only one `(show|hide)` group captured; `import 'x.dart' show A hide B;` loses the hide.
   - **Fix**: Document limitation or add second combinator group.

6. **M4. `.part.dart` missing from generated suffixes** ‚Äî `imports.py:282`
   - Riverpod `.part.dart` files not filtered. `.gr.dart`/`.config.dart` undocumented in DART_SUPPORT.md.

7. **M5. Constructor double-prefixing in qualified_name** ‚Äî `extractor.py:377-411`
   - `_extract_name` returns `"UserService.withDefaults"`, then `_extract_symbol` prepends parent ‚Üí `"UserService.UserService.withDefaults"`.
   - **Fix**: Skip parent prefix when name already contains it, or return only trailing part.

8. **M6. `method_signature` missing from `name_fields`** ‚Äî `languages.py:445-455`
   - Works via special-case in extractor but fragile; silent breakage risk.

9. **M7. `mixin_declaration` missing from `name_fields`** ‚Äî `languages.py:445-455`
   - Same issue as M6.
